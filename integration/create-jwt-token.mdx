---
title: How can I create JWT token without 2FA?
---

JWT Token creation is only available for sub-users.

**Authorisation:**

- Auth type: Basic  
- Login: sub-user email  
- Password: sub-user password

---

### Steps

1. Execute method [`POST /v2/auth/token`](/api-reference/endpoint/v2-auth-token) to get access tokens.

```bash
curl -X POST "https://api.sandbox.altpay.uk/v2/auth/token" \
  -u username:password \
  -H "Content-Type: application/json" \
  -d '{"username": "user", "password": "pass"}'
```

2. You will get `signId` in response  

3. Execute method [`POST /v2/mfa/{type}/sign/{requestId}`](/api-reference/endpoint/verify-mfa) to submit the verification code and receive a `signToken`

```bash
curl -X POST "https://api.sandbox.altpay.uk/v2/mfa/sms/sign/abc123" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <your-token>" \
  -d '{
    "code": "111111"
  }'
```

4. You will get `signToken` value

5. Execute method [`POST v2/auth/token`](/api-reference/endpoint/v2-auth-token) again with body:

```json
{
  "signToken": "<from step 3>"
}
```

6. Authorisation: Bearer Token with `accessToken` value from previous step

7. Execute [`POST v2/auth/`](/api-reference/endpoint/v2-auth) with body:

```json
{
  "id": "00000000-0000-0000-0000-0000000000120",
  "allowIps": ["127.0.0.1"],
  "withMfa": false,
  "fingerprint": null,
  "expiredAt": null,
  "app": "Chrome"
}
```

8. You'll get `signId` value

9. Execute again:  
   `POST /v2/mfa/{type}/sign/{requestId}` as in step 3

10. Get `signToken`

11. Execute final:  
    `POST /v2/auth` with body:

```json
{
  "signToken": "<from step 10>"
}
```

12. Youâ€™ll receive response:

```json
{
  "status": "CREATED",
  "id": "..."
}
```

13. Pair of `accessToken` and `refreshToken` will be sent via WebSocket

14. Use `accessToken` in:  
    `Authorization: Bearer <accessToken>`

15. Now you can use API methods with this authorisation.
